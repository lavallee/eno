# Building a Research Workflow

An active thread pulling together the pieces of a repeatable research workflow — the
path from a raw source to a durable, linked note.

## The pipeline

Capture goes to a single inbox; processing turns each keeper into an atomic
[[Zettelkasten]] note; connection links it into [[Personal Knowledge Management]]
structures; retrieval leans on [[Spaced Repetition]] and [[Active Recall]] so the
knowledge stays live. Distillation uses [[Progressive Summarization]], and clarity
checks run through [[The Feynman Technique]].

## Structuring the day

[[Time Blocking]] carves out the deep-work hours the pipeline needs, and the whole
thing lives inside a [[Digital Garden Setup]]. The [[Map of Content - Learning]] note
is the index I navigate it all from. I keep coming back to [[Systems Thinking]] to see
the workflow as a set of stocks and flows rather than a checklist. Open question: how
much of this can be handed to a [[Second Brain]] before the [[Cognitive Load]] of
maintaining the system outweighs the payoff.
